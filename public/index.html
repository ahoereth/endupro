    Heart Rate vs Pace

    Read each bubble as one pace/HR segment: right is faster pace, higher is higher heart rate. Bubble color uses shades of blue by date: lighter blue is earlier in the selected color window, darker blue is more recent. A key progress signal is at the same pace (same x-value): darker bubbles lower on the chart than lighter bubbles mean lower heart rate for the same speed.

    How To Read Metrics

    • Running Tolerance badge summarizes your current load state: Below Baseline, Near Cap, or Above Cap.
    • Load vs baseline shows percent and absolute change versus the long-term baseline, plus how far you are below/above the +10% cap.
    • 7d / 90d baseline / +10% cap: 7d is current weekly load, 90d baseline is long-term weekly baseline, and +10% cap = 1.10 * 90d baseline.
    • Monotony is mean(day km over last 7d) / stddev(day km over last 7d). Lower means more day-to-day variation; higher means a flatter, more repetitive load pattern. Zones: <=1.5 low, 1.5-2.0 moderate, >2.0 high.
    • Range Foundations summarizes the selected timeline range with distributions of run distance, pace, start time of day, weekday, and duration.
    • Heart Rate vs Pace uses interval splits first, then per-km splits, then run averages. With the reversed x-axis, right is faster pace. More negative Pearson r means faster pace is more associated with higher HR.
    • Comparable runs only filters HR-vs-pace points to runs similar to a reference run (selected run or latest run), so fitness shifts are easier to compare.
    • Pace-Bin HR Heatmap shows weekly average HR by pace bin. Drag both color handles to focus detail in your chosen HR range.
    • Activity Details opens run-level drilldown (summary, splits/intervals, and baseline context on that date).
    • Use this as training guidance, not medical advice. Combine with soreness, sleep, and injury history.

    HR Zone Settings

    Defaults come from your synced Intervals.icu running threshold HR. Override values are optional and are applied locally. A threshold override recalculates the applied zones before any per-zone overrides are layered on top.
